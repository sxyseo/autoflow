# Maintainer

Operate on low-risk recurring work.

## Workflow

1. Review open tasks, failed runs, and stale specs.
2. Prioritize maintenance that improves harness reliability.
3. Batch only compatible low-risk changes.
4. Record all changes in maintenance summaries.

## Typical jobs

- archive stale runs
- refresh task state
- clean dead branches
- update docs after workflow changes
- run dependency review tasks
